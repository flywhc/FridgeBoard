# FridgeBoard 项目约定

## 已验证命令

- 后端静态检查：`uv run ruff check backend`
- 后端测试：`uv run pytest`
- Python 锁文件检查：`uv lock --check`
- 前端静态检查：`npm run --prefix frontend lint`
- 前端生产构建：`npm run --prefix frontend build`
- 单容器镜像构建：`docker build --tag fridgeboard:local .`

## 工程边界

- 生产环境固定为单个 FastAPI/Uvicorn 进程与单个容器副本；SQLite 后续启用 WAL 后不得通过增加 worker 或副本扩展写入。
- API 与 PWA 静态产物必须同域提供；不要为首版增加跨域部署。
- 密钥仅能由部署环境注入，禁止提交 `.env`、数据库文件、令牌或生产数据。

## 文档与进度同步

- 领取实现或修复任务后、修改任何应用代码、API、数据库迁移或运行配置前，必须先将 `docs/progress-tracker.md` 对应任务更新为“进行中”，新增会话记录并写明目标、范围、设计资产/需求基线与预期验证；未完成该记录不得开始代码修改。
- 每次实现或修复任务完成后，必须及时同步 `docs/progress-tracker.md` 及受影响的功能、设计、运行或验收文档；进度状态、会话记录、验证证据和未验证项必须与实际代码保持一致，不得留下“未开始”或“待领取”等过期状态。
- 任务转为“待评审”“完成”或“阻塞”时，必须在同一会话内更新进度状态、验证证据、未验证项和下一步；不得以代码已修改或测试已通过替代文档同步。
- 若任务涉及已登记的功能包（P0–P11），至少更新进度看板对应行和会话记录；尚未满足完成条件时应标为“待评审”或“进行中”，并明确剩余验收项。

## 界面实现

- 涉及用户界面的新增或修改时，若任务、产品文档或设计资料提供了明确设计图，必须以该设计图为实现依据，并在交付前完成对应视觉核验；不得以通用组件、示意布局或自行猜测替代设计图。仅当没有明确设计资料时，才可自行提出界面方案，并应在交付中说明该前提。
- `docs/ui-design-specification.md` 是跨页面布局与样式的强制规范。所有 UI 任务在阅读具体设计稿前必须先完整阅读该规范，并按其中的页面类型选用统一页面壳、顶部栏、标题、内容区和底部操作区；禁止逐页复制或重写同语义布局。
- 设计稿负责具体场景，本规范负责共享规则。两者不一致时，共享布局和样式以本规范为准；若冲突会改变信息层级或核心交互，必须先报告“规范—设计稿冲突”并等待确认，不得自行折中。
- 品牌中文名统一为“家常食橱”。手机端一级页的品牌字标必须位于顶部栏的视口水平中心；二级页标题必须位于三列顶部栏中间；普通页面不得把品牌或标题放到右侧、内容中段或通过固定外边距/绝对定位伪造位置。
- 顶部栏、页面壳、底部操作区及同语义控件必须复用共享组件和设计令牌。页面代码不得覆盖共享顶部栏的列宽、字号和对齐方式；确需例外时必须符合规范中的例外类型，或先更新规范并说明理由。

### 最终设计图获取（跨会话必须遵循）

- 已确认的最终 UI 清单及草稿 ID 位于 `docs/final-ui-designs.md`；本地导出的 PNG、HTML 与视口信息位于 `docs/ui-assets/manifest.json`、`docs/ui-assets/png/` 和 `docs/ui-assets/html/`。功能规则和草稿 ID 索引位于 `docs/functional-design-and-feasibility.md` 的 §17.1。实现 UI 前必须依次阅读 `docs/ui-design-specification.md`、上述功能规则、设计图注册表和对应本地 PNG/HTML，并按页面/流程匹配对应草稿。
- 本地资产是实现和视觉核验的首选及充分依据；不要因远端 Preview URL 不可访问而停止实现或要求用户重新提供截图。
- 远端 `Preview URL` 仅用于对照或重新导出。若需要 Superdesign CLI 且默认沙盒网络/本机代理无法访问远端资产，必须以最小范围申请 `require_escalated` 权限，再按 Superdesign skill 验证 `npx --yes @superdesign/cli@latest --version`、登录状态并执行 `npx --yes @superdesign/cli@latest get-design --draft-id <草稿 ID> --json`。不得从历史对话猜测设计内容。
- 仅在对应本地资产不存在、且已最小范围申请外网权限后仍无法读取远端草稿时，才是外部设计资产访问阻塞。应报告对应场景、草稿 ID、URL 和失败原因；在获得设计前不得以通用布局代替。
- 实现完成后必须同时对照 UI 规范和本地 PNG 完成视觉核验（截图/浏览器检查）；在交付中注明草稿 ID、核验视口、核验结果和未解决的规范—设计稿冲突。
