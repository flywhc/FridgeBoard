# FridgeBoard UI 设计规范

状态：项目级强制规范  
版本：1.0  
更新日期：2026-07-21

## 1. 目的与适用范围

本规范统一 FridgeBoard 手机 PWA 与冰箱端显示设备的跨页面布局、视觉语言和交互基线，解决不同设计稿或实现阶段出现的品牌、标题、间距和操作区漂移。

它不替代具体场景的最终设计稿和功能规则，而是约束所有页面共享的部分。新增页面、修改既有页面、从设计稿生成代码以及视觉验收时均必须遵守。

## 2. 依据与冲突处理

按以下顺序判断实现依据：

1. `docs/functional-design-and-feasibility.md` 决定功能、信息与交互语义。
2. 本规范决定跨页面布局、样式、组件和响应式基线。
3. `docs/final-ui-designs.md` 登记的最终设计稿决定具体页面的内容编排与场景细节。
4. 现有代码只能作为实现参考，不能反向覆盖前三项。

最终设计稿与本规范的共享规则不一致时，以本规范为准；不得选择性照搬不一致的标题位置、边距或控件样式。若差异会明显改变页面信息层级或核心交互，先记录“规范—设计稿冲突”并请产品确认，不得自行折中。

## 3. 品牌与命名

- 产品中文名统一为“家常食橱”，不得写成“家常食厨”或其他变体。
- 品牌字标只表达产品身份，不替代当前页面标题。
- 同一屏幕只设置一个一级页面标题。品牌字标、冰箱名称、步骤名不得同时使用一级标题样式。
- 用户创建的冰箱名称属于上下文信息；需要切换冰箱时放在顶部栏右侧入口或标题下方，不得挤占品牌字标的视觉中心。

## 4. 基础设计令牌

### 4.1 颜色

| 语义 | 色值 | 用法 |
| --- | --- | --- |
| 纸张背景 | `#F2F2EE` | 页面默认背景 |
| 内容背景 | `#FFFFFF` | 卡片、输入框、二维码底色 |
| 主文字/主边框 | `#111111` | 标题、正文、主要按钮、轮廓 |
| 次级文字 | `#777777` | 辅助说明、时间、弱化信息 |
| 分隔线 | `#D7D7D2` | 列表分隔和非选中边框 |
| 警告强调（仅手机端） | `#B6402B` | 错误或危险操作的渐进增强 |
| 正常强调（仅手机端） | `#4F6653` | 成功或新鲜状态的渐进增强 |

状态含义必须在黑白或一位色显示下仍可识别。颜色不能作为唯一信息载体；必须同时使用文字、图形、边框粗细或纹理。

### 4.2 字体

- 字体栈统一为 `Arial, "Helvetica Neue", sans-serif`，数字与日期使用等宽数字特性。
- 手机端品牌字标：12–13px、700、字距 `0.16em`。
- 手机端内容页一级标题：28px、700；顶部导航栏标题：16–18px、700。
- 手机端分组标题：18px、700；正文和表单控件不得小于 16px；辅助文字不得小于 12px。
- 冰箱端页面标题：30–36px、700；关键数量：24–32px、700；标签：18–22px；辅助信息：16–18px。
- 标题允许紧凑字距，但不得通过绝对定位或负边距制造视觉居中。

### 4.3 间距、边框与圆角

- 以 8px 为基础间距单位；常用间距只使用 4、8、12、16、24、32、40、48px。
- 手机端水平安全边距为 16px；冰箱端为 20–24px。
- 手机端交互热区最小 48×48px；冰箱端最小 56×56px。
- 默认边框 2px；选中或警告 3px；过期或破坏性强调 4px。
- 冰箱端圆角 0–4px；手机端控件圆角 8–12px。
- 不使用阴影、玻璃效果、模糊背景、渐变或仅装饰性的纹理。斜纹仅用于确定性的状态表达。

## 5. 页面骨架

页面只允许由以下区域按顺序组成：

```text
系统安全区
顶部栏（品牌栏或页面导航栏）
页面标题/上下文（仅在该页面类型需要时）
可滚动主内容区
固定或随内容排列的主要操作区
底部安全区
```

- 顶部栏必须位于页面顶部并参与正常文档流；允许 `sticky`，不得用任意 `top` 值或垂直居中容器定位。
- 普通页面不得用 `justify-content: center` 垂直居中整个页面。只有首次开机、配对成功、空结果等明确的单任务状态页可以垂直居中主体；其顶部品牌仍保持在顶端。
- 页面根容器至少占满 `100dvh`，并处理 `env(safe-area-inset-top)` 与 `env(safe-area-inset-bottom)`。
- 长内容只滚动主内容区。顶部栏和固定操作区不得覆盖标题、表单末项或反馈信息。

## 6. 顶部栏与标题

### 6.1 手机端一级页面

适用于当前冰箱、食谱、补货、我的冰箱等主入口：

- 顶部使用品牌栏，品牌字标“家常食橱”在**视口水平中心**，不以剩余空间为中心。
- 顶部栏采用三列网格：`48px minmax(0, 1fr) 48px`。左、右入口即使缺失也保留等宽占位。
- 左侧放全局导航入口；右侧放冰箱切换、设置或该页唯一的高频操作。
- 页面业务标题如需显示，放在品牌栏下方的内容区并左对齐，不得把品牌字标推到右侧或屏幕中部。

### 6.2 手机端二级页面与流程页

适用于编辑食材、布局编辑、提醒设置、单日食谱编辑等：

- 顶部使用页面导航栏：左侧返回，中间当前页面标题，右侧放至多一个文字或图标操作。
- 导航栏同样采用 `48px minmax(0, 1fr) 48px` 三列网格。标题在视口水平中心，左右内容不等宽时也不得偏移。
- 返回、标题和右侧操作垂直居中；标题单行截断，不换行挤高顶部栏。
- 不在导航栏中再次显示大号品牌字标。需要品牌识别时使用应用壳或启动页，不与页面标题争夺层级。

### 6.3 冰箱端页面

- 首页顶部固定展示当前页面标题、库存/同步上下文和刷新入口；不得把标题放进冰箱图形内部。
- 分区详情、补货清单和配对页使用三列顶部栏：返回、居中标题/区域缩略图、刷新或状态入口。
- 冰箱端顶部栏热区不小于 56px；禁止悬停、滑动和长按依赖。
- 页面标题和操作在滚动时保持可见；无滚动页面也必须占据顶部固定区域。

### 6.4 允许的例外

- 相机/扫码页可以使用深色沉浸式顶部栏，但品牌或页面标题仍位于顶部，不得漂浮在取景框中。
- 首次开机、配对成功、加载完成等状态页可垂直居中主体插图和结果文案；品牌栏仍固定在顶部安全区之后。
- 全屏确认弹层可不显示品牌栏，但必须有明确标题和关闭/返回入口。

## 7. 内容、表单与操作区

- 手机端主内容以单列为默认；相邻区块垂直间距 24px，区块内元素间距 8–16px。
- 卡片只用于确有分组语义的内容，不把每段文字都装入卡片。列表优先使用分隔线保持信息密度。
- 表单标签置于控件上方，必填与可选状态明确；输入、选择和按钮高度至少 48px。
- 每屏只能有一个主操作。次操作使用描边或文字样式，危险操作不得与主操作同色同权重。
- 固定底部操作区必须包含底部安全区内边距，并为主内容预留等高空间。
- 加载、空数据、错误、禁用、离线、长文本和最窄 320px 宽度都必须有明确布局，不得依赖内容恰好较短。

## 8. 图标与状态

- 图标使用 24×24 或 32×32 网格、2–3px 描边，并保持同一端内的线宽一致。
- 图标按钮必须提供可访问名称；图标本身不等于点击热区。
- 新鲜为普通轮廓；临期为双边框与斜纹角标；过期为黑底感叹号。未填写 BBD 不显示异常状态。
- 同一状态在冰箱端与手机端使用同一符号语义，不得逐页发明新徽记。

## 9. 响应式基线

- 手机端设计基准为 390×844px，同时验证 320、390、430px 宽度；页面宽度流式适配，不锁死 390px。
- 手机端宽屏时内容容器居中，但顶部栏的视觉中心仍以容器中心为准。
- 冰箱端按约 3:4 竖屏比例设计，至少验证 540×720px；不得写死 Kindle 型号或单一分辨率。
- 禁止使用依赖固定内容高度的巨大 `margin-top`、负边距或绝对定位来摆放标题和主要操作。

## 10. 共享组件与实现约束

- 顶部栏、页面壳、底部操作区、按钮、表单控件、状态徽记和卡片必须复用共享实现，不得在各页面复制一套略有差异的 CSS。
- 页面级组件只配置标题、返回行为、右侧操作和是否固定；不得自行覆盖共享顶部栏的网格列宽、字号和对齐方式。
- 颜色、字体、间距、圆角、边框和热区尺寸应定义为设计令牌；业务组件不得散落魔法数。
- 新增例外前先确认能否归入第 6.4 节；确需新增时，必须同步更新本规范和共享组件测试。

建议共享接口至少覆盖以下语义，不强制具体命名：

```text
PageShell(surface, header, footer, scrollMode)
AppHeader(leftAction, wordmark, rightAction)
PageHeader(backAction, title, rightAction)
BottomActionBar(primaryAction, secondaryAction)
```

## 11. 视觉核验与交付门槛

每个 UI 任务交付前必须完成：

1. 记录对应场景、最终草稿 ID 和预览 URL。
2. 对照本规范检查页面类型、顶部栏、标题层级、边距、控件和状态语义。
3. 手机端至少截图核验 390×844px，并检查 320px 与 430px 宽度；冰箱端至少截图核验一个 3:4 竖屏视口。
4. 居中标题的几何中心与内容容器中心误差不得超过 2px；不能只凭肉眼判断“看起来差不多”。
5. 检查长标题、系统字体放大、加载、空、错误、离线和禁用状态。
6. 在交付说明中列出已核验视口、结果和仍存在的设计稿冲突。

出现以下任一情况不得声明 UI 完成：

- 品牌字标或页面标题不在规定的顶部区域。
- 应居中的标题受左右按钮宽度影响而偏移。
- 通过固定高度、巨大外边距或绝对坐标把标题“摆到”目标位置。
- 页面之间同语义控件的字号、边框、圆角或热区不同。
- 无法访问设计稿且没有用户提供的截图或结构说明。
