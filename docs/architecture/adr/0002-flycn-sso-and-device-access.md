# ADR-0002：复用 flycn 用户身份与设备访问模型

状态：已接受  
日期：2026-07-19

## 决策

`app.flycn.fyi` 是 FridgeBoard 的唯一账号和密码身份源。FridgeBoard 通过 SSO 回跳、60 秒单次授权码和私网兑换接口建立自己的持久管理会话；不读取 flycn SQLite，不共享 Cookie、签名密钥或密码哈希。

任意 flycn 活跃用户均可成为冰箱所有者。每台冰箱只有一个所有者。未登录手机通过 Kindle 的二维码取得独立、可撤销设备凭证，并可执行日常库存和食谱操作；所有者专属操作仅限已登录管理会话。

## Kindle 首次绑定与手机配对

1. 已登录手机创建 6 位数字 Passcode，指定“新建冰箱”或一台已有冰箱。
2. Kindle 在首页输入 Passcode；服务端消费后创建/绑定冰箱并发放 Kindle 设备凭证。
3. Kindle 保存设备身份并显示 10 分钟、单次使用的手机配对二维码。
4. 手机 PWA 扫码后取得自己的设备凭证；二维码随使用或过期自动关闭。

## 后果

- Passcode 与二维码均只在服务端保存哈希和值的状态，不作为长期访问凭证。
- 管理会话为长期持久会话；flycn 后续的改密码、改角色或禁用状态仅在下次 FridgeBoard 登录时生效。这是为当前低安全、家庭使用场景选择的简化取舍。
- 清除浏览器数据、撤销设备或删除冰箱后，相关设备须重新配对。
